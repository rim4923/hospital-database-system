# Hospital Database Management System

## Overview

This project designs and implements a relational database system for managing hospital operations, including patients, doctors, services, and inventory.

The system focuses on efficient data organization, integrity, and real-world healthcare workflows.

---

## Features

* Entity-Relationship (ER) modeling
* Relational schema design
* Normalization up to BCNF
* SQL queries for real-world operations

---

## System Design

The database includes key entities such as:

* Patients
* Doctors
* Staff
* Departments
* Medical Records
* Services
* Inventory
* Suppliers
* Insurance

Relationships were carefully designed to reflect real-world hospital workflows.

---

## Database Concepts Applied

* ER → Relational mapping
* Primary and foreign keys
* Functional dependencies
* Normalization (1NF → BCNF)
* Query optimization

---

## Example Queries

* Retrieve available doctors by specialty
* Track inventory shortages
* Identify patients without insurance
* Analyze supplier performance
* Monitor emergency resources
